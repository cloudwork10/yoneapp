# خطوات رفع الباك اند (YONE API) على السيرفر

هذا الملف يشرح كيف تشغّل الباك اند على جهازك أولاً، ثم كيف ترفعه على سيرفر حقيقي حتى التطبيق يقدر يتصل به من أي مكان.

---

## الجزء ١: تشغيل الباك اند على جهازك (للتجربة)

### ١. تثبيت Node.js
- حمّل من: https://nodejs.org (النسخة LTS)
- ثبّت وافتح التيرمنال وتأكد:
```bash
node -v
npm -v
```

### ٢. تشغيل MongoDB (قاعدة البيانات)
- الباك اند محتاج MongoDB. إما:
  - **على جهازك:** ثبّت MongoDB من https://www.mongodb.com/try/download/community وشفّنه، أو
  - **سحابة مجانية:** سجّل في https://www.mongodb.com/cloud/atlas وأنشئ قاعدة مجانية وانسخ "Connection String"

### ٣. إعداد ملف البيئة (Environment)
- من داخل مجلد المشروع ادخل لمجلد الباك اند:
```bash
cd backend
```
- انسخ ملف مثال البيئة وعدّل القيم:
```bash
cp config.env.example config.env
```
- افتح `config.env` بمحرر نصوص وعدّل:
  - `DB_CONNECTION_STRING` = رابط MongoDB (مثلاً من Atlas أو `mongodb://localhost:27017/yoneapp`)
  - `JWT_SECRET` = أي نص طويل وعشوائي (مهم لا تشاركه)
  - باقي القيم ممكن تتركها للتجربة

### ٤. تثبيت المكتبات وتشغيل السيرفر
```bash
cd backend
npm install
node server.js
```
- لو ظهرت رسالة مثل: `YONE API Server started on port 3000` يبقى الباك اند شغال على جهازك.
- جرّب من المتصفح: `http://localhost:3000/api/health` لازم يرجع JSON فيه "status": "success".

---

## الجزء ٢: رفع الباك اند على سيرفر حقيقي (عشان التطبيق يتصل من برة)

معنى "الرفع": إنك تشغّل نفس الباك اند على سيرفر (كمبيوتر) متصل بالإنترنت ٢٤ ساعة، وتاخد له عنوان (دومين أو IP) عشان التطبيق يرسل الطلبات عليه.

### خيار أ: استضافة جاهزة (أسهل – بدون إدارة سيرفر)

#### ١) Railway (مجاني لفترة ثم مدفوع)
1. ادخل https://railway.app وسجّل.
2. New Project → Deploy from GitHub (اربط ريبو المشروع).
3. اختر مجلد `backend` كمصدر (أو حدد أن الـ root هو backend).
4. أضف متغيرات البيئة (Environment Variables) في لوحة Railway:
   - `NODE_ENV` = production
   - `DB_CONNECTION_STRING` = رابط MongoDB (استخدم Atlas)
   - `JWT_SECRET` = نفس السر الطويل
   - `BASE_URL` = رابط الـ API اللي هيعطيك Railway (مثل https://xxx.railway.app)
   - `ALLOWED_ORIGINS` = رابط التطبيق أو الموقع إن وُجد (مثلاً https://yourapp.com)
   - بقية متغيرات Paymob لو عندك
5. احفظ وانتظر الـ deploy. Railway يعطيك رابط مثل: `https://yoneapp-backend.railway.app`
6. في التطبيق (في ملف `.env` في جذر المشروع):  
   `EXPO_PUBLIC_API_URL=https://yoneapp-backend.railway.app`

#### ٢) Render (مجاني tier)
1. ادخل https://render.com وسجّل.
2. New → Web Service.
3. اربط الريبو واختر المشروع.
4. Build command: `cd backend && npm install`
5. Start command: `cd backend && node server.js`
6. في Settings أضف Environment Variables (نفس القيم اللي فوق).
7. احفظ وانتظر الـ deploy. Render يعطيك رابط مثل: `https://yoneapp-api.onrender.com`
8. في التطبيق: `EXPO_PUBLIC_API_URL=https://yoneapp-api.onrender.com`

#### ٣) MongoDB Atlas (قاعدة البيانات في السحابة)
- سجّل في https://www.mongodb.com/cloud/atlas
- أنشئ Cluster مجاني → Connect → "Connect your application" → انسخ الـ Connection String
- ضع اسم المستخدم وكلمة المرور في الرابط واستخدمه في `DB_CONNECTION_STRING` في Railway أو Render

---

### خيار ب: سيرفر خاص بك (VPS مثل DigitalOcean أو استضافة عربية)

1. **اشتري سيرفر** (أبسط خطة تكفي).
2. **اتصل بالسيرفر** عبر SSH.
3. **ثبّت Node.js و MongoDB** (أو استخدم MongoDB Atlas وربطه بالسيرفر).
4. **انقل ملفات الباك اند** (مثلاً بـ git clone للمشروع).
5. **أنشئ ملف `config.env` أو `.env`** بنفس المتغيرات اللي فوق، مع:
   - `BASE_URL` = https://api.yourdomain.com (بعد ما تربط الدومين)
   - `ALLOWED_ORIGINS` = روابط التطبيق/الموقع
6. **شغّل السيرفر بشكل دائم** بأداة مثل PM2:
```bash
npm install -g pm2
cd backend
pm2 start server.js --name yone-api
pm2 save
pm2 startup
```
7. **ربط دومين و SSL (HTTPS):** استخدم Nginx كـ reverse proxy و Let's Encrypt (شهادات مجانية). هذا يحتاج خطوة إضافية يمكن شرحها لاحقاً.

---

## ملخص المتغيرات المهمة للإنتاج

| المتغير | معناها | مثال |
|---------|--------|------|
| NODE_ENV | بيئة التشغيل | production |
| DB_CONNECTION_STRING | رابط MongoDB | من Atlas أو السيرفر |
| JWT_SECRET | سر التوقيع | نص طويل عشوائي |
| BASE_URL | عنوان الباك اند (لروابط الصور/الفيديو) | https://api.yourdomain.com |
| ALLOWED_ORIGINS | من يسمح له بطلب الـ API | https://yourapp.com |
| PAYMOB_* | إعدادات Paymob | من لوحة Paymob |

---

## بعد الرفع

1. جرّب من المتصفح: `https://رابط-الباك-اند/api/health`
2. في التطبيق ضع في `.env`:  
   `EXPO_PUBLIC_API_URL=https://رابط-الباك-اند`
3. أعد بناء التطبيق أو شغّله من جديد عشان يقرأ العنوان الجديد.

لو واجهت خطوة معينة (مثلاً Railway أو Render أو PM2) اذكرها ونشرحها بالتفصيل.
